Code Reference

Welcome to the NodeBox technical reference. Here you'll find a high-level overview, module documentation, and API details to help you extend or debug the project.

Architecture Overview

A brief overview of the main components of the NodeBox application.

Key Modules

This section outlines the primary folders in the source code and their general purpose.

automation_manager
Purpose: Contains the core logic and UI for creating and editing automation nodes and their connections.
browsemodels_manager
Purpose: Manages the UI and logic for browsing, downloading, and viewing local Ollama models.
canvasmanager
Purpose: Handles the main visual canvas, including drawing nodes, ports, and connections, as well as saving/loading workflows.
features
Purpose: Contains additional application features like the debug console, performance monitor, and workflow scheduler.

Class & Function Reference

A more detailed look at important classes and functions within the key modules.

main.py
Purpose: This is the main entry point that starts the entire NodeBox application. It initializes the main window and begins the application event loop.
automation_manager/nodeeditor_dialog.py
Class: NodeEditorDialog(QDialog)
Purpose: This class builds the responsive dialog window where users can write Python code for a specific node. It uses a `QSplitter` layout for flexibility and handles the logic for running code and saving results.
canvasmanager/canvas_widget.py
Class: CanvasWidget
Purpose: This is the core UI component for the visual workflow canvas. It handles all mouse events for dragging nodes, creating connections, and navigating the canvas.
